# Implementation Language Rules

Use this reference when a presentation enters scheme design, detailed design, renderings, product form, construction logic, or any page where the proposal must become tangible.

## Phase shift

Concept and strategy pages may answer "why". Implementation pages must answer "what is used, where it is placed, and what it changes".

Use this sentence chain:

`visible/professional object -> design action -> user-facing effect -> summary value`

The summary value is mandatory on implementation pages. It should answer one of these questions in audience-facing language:

- What problem does this page solve?
- What use or operation does it support?
- Why is this design action necessary?
- What does it change for the guest, user, client, brand, or decision maker?

Examples:

- `镜面水景和入口景墙先把视线收住，再用植物和片墙挡掉外围干扰，让客人进入花园时先看到酒店自己的界面。`
- `深色石材界面、木质顶面和暖光共同压低空间情绪，让会客区显得更稳，也更适合商务接待。`
- `白色曲面矮墙和艺术绿篱弱化硬边界，草坪边缘因此更柔和，停留感也更偏度假。`
- `餐厅界面、外摆座椅和暖色灯光共同把室内功能延伸到花园里，让这里不再只是窗外绿化，而是酒店可以日常经营的半室外会客场。`
- `镜面水景和入口景墙把第一眼视线收住，解决原来入口一眼望穿的问题，也让客人进入花园时先获得一个明确的停顿点。`

## Concrete anchors

When visible or stated in the source, prefer concrete anchors over abstract concept words.

Landscape anchors:

- paving, lawn, water feature, stepping stone, retaining edge, low wall, feature wall
- pergola, canopy, gray space, bar counter, outdoor seating, furniture
- hedge, tree canopy, shrub massing, planter, planting screen
- lighting, warm light, linear light, sign wall, glass interface
- circulation, entrance view, framed view, transition, enclosure, open lawn

Interior anchors:

- ceiling, wall finish, stone, terrazzo, wood veneer, metal trim, glass
- counter, cabinet, soft seating, lighting, display shelf, screen, door opening
- circulation, waiting area, meeting area, privacy line, acoustic or visual boundary

Industrial/product anchors:

- shell, grip, hinge, button, display, module, texture, interface, assembly seam
- material, process, tolerance, weight, radius, maintenance, replacement path

Public-sector/report anchors:

- responsible unit, service window, route, time node, data interface, public counter
- review process, response mechanism, risk point, duty boundary, measurable result

## Do not over-invent

Concrete does not mean fabricated.

- If the PDF names a material, use that exact material.
- If the material is only visible but not named, use cautious words such as `深色石材意向`, `木质感界面`, `白色曲面墙体`, or `金属线条感`.
- If the material or process cannot be safely inferred, write the visible object and mark the page `needs-user-edit`.
- Do not invent construction methods, brands, costs, approvals, or final material specifications.

## Voice

Implementation language should sound more direct than concept language.

Prefer:

- `用镜面水景把入口压静。`
- `用绿篱和片墙挡掉外围杂乱视线。`
- `吧台、外摆桌椅和暖光让花园具备日常经营条件。`
- `曲面矮墙把草坪边界做软，减少屋顶空间的生硬感。`

Avoid:

- `形成高级空间气质。`
- `营造东方美学体验。`
- `延续概念并完成空间升华。`
- `体现项目独特价值。`

## Summary sentence

Do not stop after naming visible objects. Add one short summary sentence or clause that makes the page useful for reporting.

Weak:

- `餐厅界面、外摆座椅和暖色灯光共同把屋顶花园变成可经营场景。`

Better:

- `餐厅界面、外摆座椅和暖色灯光共同把屋顶花园变成可经营场景，让客人从室内望出去时看到完整的酒店界面，而不是一片孤立绿化。`

Best when the PDF supports it:

- `餐厅界面、外摆座椅和暖色灯光共同把屋顶花园变成可经营场景，让客人从室内望出去时看到完整的酒店界面，也让酒店在晚间会客、轻餐和酒吧运营上有更明确的使用抓手。`

Keep the summary specific. Avoid generic endings such as `提升整体品质`, `增强空间体验`, or `体现设计价值` unless the sentence has already named the concrete problem and effect.

## Repeated rendering groups

For consecutive effect-image pages:

1. First page: name the real space and the main tangible anchors.
2. Middle pages: mention only the new visible difference, such as material, viewpoint, lighting, furniture, planting, or edge treatment.
3. Later pages with no new information: use a short support sentence and a quick-flip cue.

Do not keep repeating concept words across rendering pages. A rendering page should feel like the presenter is standing in the space and naming what is physically making the experience happen.
