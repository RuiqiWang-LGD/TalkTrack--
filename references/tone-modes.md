# Tone Modes

Use this reference before drafting when the user has not specified a speaking style, or when the presentation needs a different level of professionalism, emotional value, or decision focus.

Ask the user to choose one mode when time allows. If there is no answer, default to `直读稳妥型`.

## Mode 1: 直读稳妥型

Best for exhausted presenters, temporary substitutions, leaders reporting several projects, or any case where the speaker may not know the project.

Principle:

- Highest safety.
- Fewest invented details.
- Smooth spoken rhythm.
- Clear subject, clear cause and effect.
- Suitable for direct reading with almost no preparation.

Use when:

- The user says "拿起来直接念", "没时间准备", "明天就汇报", or "先完成再完美".
- The audience is mixed and professional depth is unknown.
- The source PDF has limited text and many images.

Language:

- Plain but formal.
- Short sentences.
- Strong chapter connection.
- Avoid decorative value claims.

## Mode 2: 专业落地型

Best for design review, technical discussion, professional clients, internal design review, or pages that show actual deliverables.

Principle:

- More tangible objects.
- More discipline terms.
- More material, node, structure, process, and function words.
- Less abstract concept language once the presentation reaches implementation.

Use when:

- The user wants the middle and later sections to feel more professional.
- The PDF contains plans, sections, renderings, details, materials, product structure, or service process.
- The presenter is a designer who can supplement real project facts live.

Language chain:

`objective fact or visible object -> professional action -> user-facing effect -> one summary sentence`

Example:

`镜面水景先把入口压静，入口景墙和植物负责遮挡外围干扰，室外吧台和外摆家具再把这里变成可经营的停留点。`

## Mode 3: 价值叙事型

Best for proposal, brand presentation, public-facing pitch, video script, and moments where the project needs emotional lift.

Principle:

- More narrative and value framing.
- More inclusive, emotional, memorable wording.
- Still grounded in visible facts.
- Do not let poetic language replace project logic.

Use when:

- The user wants the presentation to sound more "高级", more emotional, or easier to promote.
- The audience needs to feel the project value before judging details.
- The output may be reused for short videos, social media, or sales presentation.

Language:

- Use scene, feeling, memory point, user relationship, city relationship, brand relationship.
- Keep one concrete anchor in every paragraph.
- Avoid empty grandeur such as "赋能", "升华", "极致价值" unless the PDF supports it.

## Mode 4: 决策汇报型

Best for owners, senior leaders, government meetings, investment decision, project steering meetings, and time-limited reviews.

Principle:

- State problem, response, value, and decision point.
- Reduce decorative explanation.
- Emphasize tradeoffs, risks, deliverables, and next actions.
- Sound calm, responsible, and easy to approve.

Use when:

- The user says the audience is a leader, mayor, owner, boss, review committee, or non-design decision maker.
- The presentation needs to help people make a choice rather than appreciate details.
- There are multiple schemes to compare.

Language:

- "The issue is..."
- "The response is..."
- "The value is..."
- "The difference between schemes is..."
- "The next decision is..."

## Choosing the mode

If the user gives no mode:

1. Use `直读稳妥型` for opening, background, and general analysis.
2. Use `专业落地型` automatically when the PDF reaches deliverables, master plans, renderings, details, product structures, or execution pages.
3. Use small amounts of `价值叙事型` only in openings, concept pages, and closing pages.
4. Use `决策汇报型` when the audience or page clearly involves scheme comparison, approval, investment, risk, or next-step confirmation.

Do not mix all modes evenly. Pick one dominant mode, then apply phase-specific adjustments.

When confirming a new source file, recommend one dominant mode and add one short adjustment sentence. Make it clear that the recommendation can be changed before generation, for example to become more emotional, more professional, more direct-read, or more decision-oriented. Do not list all modes unless the user asks.
